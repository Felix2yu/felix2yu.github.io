Firefox or Chrome

发表于 2012-09-21 | 更新于 2018-04-25 | 分类于 科技 | 阅读次数：

这个问题困扰了我两年，到底是选择 Firefox 还是 Chrome。

##Firefox

####优势
Firefox 的优势在于，扩展性、可制定性强。

基于强大的 XUL ，Firefox 可以开发出强大的功能，基本是只有你想不到的没有实现不了的。

火狐界面可制定性极高，可以改得你完全不知道这是什么软件。

拥有数量非常大、质量很高的扩展。

Android 版可以把手机上的标签页直接推送到其他设备。

####劣势
但强大的功能同时带来了臃肿、效率低等问题。

开发较慢，而且经常会做一些匪夷所思的决定。

由于这两年 Chrome 极其强势，大量 Firefox 用户和扩展开发者都投奔 Chrome ，导致近 2 年，扩展质量大幅下降，很多 4.0 之前的经典、好用扩展都没人维护。

不支持iOS 平台。

###Chrome

####优势
有谷歌这个有钱的好爸爸，开发相当给力。

速度快、安全、有各种新奇的功能。

有完善的谷歌系统支持，对于我这种谷歌服务重度用户来说非常方便。

多设备云同步，对于我这种多平台用户来说相当赞。

现在扩展非常多，优秀扩展也越来越多。

很多 Android 应用的 PC 端的扩展都只支持 Chrome。

####劣势
基础功能缺失，比如：

  • 标签页设置（自定义打开标签页逻辑等）
  • 没有像样的恢复已关闭标签页的扩展
  • 可制定性极低
  • 对 UserScript 支持很差
  • 页面放大效果差

其实吧，只要能有个好用的标签页设置（类似于 Firefox 的 TU & TMP这种），再不济只要有个允许单击连接后台打开、新打开标签页紧临当前标签页这俩我说不定就转投 Chrome 门下了。

在上个月我换了两天的 Ubuntu，软件缺失很严重，但是因为我 Firefox 这一套扩展都在，所以跟网络相关的工具（文件、图片、视频下载）都不需要太担心。

目前在强大的功能和速度上我选择了火狐，但要是明年火狐还这样，那 Chrome 的优势，特别是谷歌帐户系统会最终会让我叛逃。
